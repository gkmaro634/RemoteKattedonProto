# ぷるぷるゲンゲ Flutter移植設計書

## 🎮 1. ゲーム概要

- **タイトル**: ぷるぷるゲンゲ
- **ジャンル**: クリック（タップ）ゲーム
- **目的**: 15秒間で中央のキャラクター「ゲンゲ」を連打してスコアを稼ぐ
- **プラットフォーム**: 現状はデスクトップ（pygame）→ Flutter でモバイル/Web へ展開予定

---

## 🛠️ 2. 主な機能・構成

### 2.1 画面構成
- **背景画像** (`umi.jpg`) を全画面に表示
- **ゲンゲ画像** (`genge.png`) を画面中央に表示、押下時および揺れ演出でサイズを変化
- **UI領域**（画面上部）に以下を表示
  - 現在スコア
  - 残り時間
  - 最高記録（右寄せ）
- **ゲーム終了時オーバーレイ**
  - 黒半透明バナー
  - ランキング結果（称号）
  - 「もう一度ぷるぷる」ボタン

### 2.2 サウンド
- クリック時効果音 (`pochi.mp3`)
- 結果音声
  - ハイスコア以上：`hakushu.mp3`
  - スコア1以上：`koto.mp3`
  - ゼロ：`gaan.mp3`
- 音声は `pygame.mixer.Sound` で再生

### 2.3 ハイスコア保存
- ファイル `highscore.txt` に整数を読み書き
- 起動時に読み込み、スコア更新時に上書き
- 例外処理あり

### 2.4 タイマー
- 制限時間 15 秒
- Pygame の `get_ticks()` で経過ミリ秒を取得し秒数を計算
- 残り時間を UI に表示し、0 になるとゲーム終了状態へ遷移

### 2.5 入力処理
- マウスクリック（Flutter ではタップ）で判定
- ゲンゲ画像の矩形 (`Rect`) に当たればスコア＋1、揺れ・エフェクト発生
- 時間切れ後はリトライボタン判定

### 2.6 エフェクト
- クリック位置から放射状に散るパーティクル（円）
- 各パーティクルは位置、速度、寿命の配列で管理
- 毎フレーム更新・描画、寿命切れで削除

### 2.7 揺れ演出
- クリック時 `is_shaking = 15`（フレーム数）
- フレームごとに減算
- ゲンゲ表示サイズに `is_shaking` を反映して拡大縮小

### 2.8 判定と称号
スコアに応じた称号を返す関数:

```python
def get_title(s):
    if s == 0: return "ただの干物"
    if s < 50: return "ぷるぷる初心者"
    if s < 100: return "コラーゲン職人"
    return "伝説のゲンゲマスター"
```

## ⚙️ 3. 処理フロー（疑似シーケンス）

1. 初期化
   - Pygame, Mixer, 画面、フォント設定
   - 画像・音声ロード
   - スコア等変数 `reset_game()` で初期化
2. メインループ（1/60 秒毎）
   - 背景描画
   - 経過時間計算 → `time_left`
   - ゲンゲサイズ調整・描画
   - イベント処理
     - QUIT で終了
     - MOUSEBUTTONDOWN:
       - 時間内＆ゲンゲクリック → スコア、揺れ、音、パーティクル発生
       - 時間切れ＆リトライクリック → リセット
   - 揺れ値・パーティクル更新
   - UI 描画（スコア・時間・ハイスコア）
   - 時間切れ時の終了処理
     - ハイスコア更新保存
     - 結果音再生（1回のみ）
     - オーバーレイ表示・判定テキスト・ボタン描画
   - 画面更新 `pygame.display.flip()`、フレームレート維持

## 📐 4. Flutter への移植ポイント

- **画面遷移** → 単一 `StatefulWidget` で実現
- **タイマー** → `Ticker`/`AnimationController` / `Timer.periodic`
- **入力** → `GestureDetector` の `onTapDown` で座標取得
- **描画** → `CustomPainter` で背景・ゲンゲ・UI・エフェクトを描画
- **リソース** → `assets/` に画像・音声を格納し `pubspec.yaml` で指定
- **サウンド再生** → `audioplayers` や `just_audio`
- **永続化** → `shared_preferences` もしくは `path_provider` + ファイル I/O
- **パーティクル** → リスト管理 + `setState` で再描画
- **揺れ演出** → ゲンゲの表示サイズを `Animation` によるスケール変化
